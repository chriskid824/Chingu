## ⏳ 仍未處理(上架前必辦)

## 1. `users` 全欄位可讀 → 收緊

現狀:`allow read: if isAuthenticated()` — 任何登入者可讀所有人完整檔案(email、生日、偏好),違反個資最小揭露,也讓「部分解鎖只給星座/產業/年齡段」形同虛設(繞過 UI 直接查 API 就能拿全部)。

建議做法(擇一):
- **公開輪廓子文件**:`users/{uid}/public/profile` 只放暱稱/星座/產業/年齡段/頭像,他人只能讀這個;主文件 `allow read: if isOwner(userId)`
- 或 Cloud Function 代查:飯友輪廓由 function 組裝回傳,users 主文件鎖本人

影響面:配對顯示、聊天頭像、評價頁名單都讀 users — 改之前 grep `collection('users')` 於 lib/ 盤點所有讀取點。

## 2. `isDeveloper()` email 後門

現狀:rules 內寫死 `chriskid824@gmail.com`,email 是可偽造感低但仍屬硬編碼身分;且 Seeder 依賴它。

建議:改 **custom claim**(`request.auth.token.admin == true`),用 Admin SDK 一次性腳本設定;或上線版 rules 直接移除 isDeveloper 分支(Seeder 只在開發環境用)。搭配環境分離(dev/prod 兩個 Firebase 專案)最乾淨。
