# 資料模型:Firestore Collections 與 Security Rules

**集合名稱一律 snake_case(bc12e5d 統一過);Dart Model 在 `lib/models/`,權限在 `firestore.rules`。**

## Collections 總表

| Collection | Dart Model | 用途 | 關鍵權限(firestore.rules) |
|---|---|---|---|
| `users` | user_model.dart | 用戶資料、飲食偏好、預算 | 登入者可讀全部(**上線前要收緊**);本人可寫 |
| `dinner_events` | dinner_event_model.dart | 每週系統級活動 | 登入者可讀;update 開放(報名寫 signedUpUsers);create/delete 僅開發者 |
| `dinner_groups` | dinner_group_model.dart | 5~7 人分桌結果 | **僅 participantIds 內成員可讀**;create/delete 僅開發者 |
| `chat_rooms` | (chat_message_model.dart) | 群組聊天 + 一對一 | 參與者可讀寫;登入者可建立(雙向 Match 時由 App 建) |
| `chat_rooms/{id}/messages` | chat_message_model.dart | 訊息**子集合**(a721954 從頂層遷移) | 經 get() 驗證 participantIds;update 一律禁止 |
| `dinner_reviews` | dinner_review_model.dart | 雙盲評價 | **只有 reviewerId 本人可讀自己寫的**;不可修改 |
| `icebreaker_questions` | — | 破冰話題 | 登入者可讀;僅開發者寫 |
| `restaurants` | restaurant_model.dart | 餐廳(名稱/地址/預算/飲食標籤) | 登入者可讀;僅開發者寫 |
| `reports` | report_model.dart | 檢舉 | 用戶只能 create;讀取僅後台 |
| `notifications` | notification_model.dart | 通知 | 本人可讀/更新;**create 僅 Cloud Functions** |
| `subscriptions` | subscription_model.dart | 訂閱(MVP 未啟用) | 本人可讀寫 |

其他 model:`booking_task_model.dart`(訂位任務,配合 bookWithValidation)。

## 雙盲設計在 Rules 層的體現

- `dinner_reviews` 的 read 條件是 `resource.data.reviewerId == request.auth.uid` — **連查詢別人給你的評價都做不到**,雙盲是資料庫層保證,不只是 UI 隱藏。
- 曾因 Seeder 查詢違反此規則導致 permission-denied(commit 2438bd9)— 寫任何查詢前先想「這查詢會不會需要讀別人的 review」。

## 已知技術債

- `users` 全欄位可讀 — rules 內註解明言「上線前收緊:只允許讀取有限欄位」
- `dinner_events` / `dinner_groups` 的 update 只檢查登入,未限制欄位 — 惡意用戶可改他桌資料,上線前要用 `request.resource.data.diff()` 限制

## AI 檢索與驗證

- 集合名稱:grep `collection('` 於 `lib/services/firestore_service.dart`
- 權限現狀:直接讀 `firestore.rules`(140 行,是最權威的資料模型文件)
- Model 欄位:讀 `lib/models/*.dart`,欄位名以 model 的 `fromJson/toJson` 為準
