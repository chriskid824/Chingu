# 聊天與解鎖機制

**兩種聊天室、兩種解鎖條件;權限由 firestore.rules 的 participantIds 保證。**

## 聊天室類型

| 類型 | 建立時機 | 參與者 | 時效 |
|---|---|---|---|
| 群組聊天 | 週三 17:00 完全解鎖時**自動建立** | 同桌全部人 | 永久保留 |
| 一對一私訊 | 雙盲互評、雙向 👍 Match 後**即時建立** | 兩人 | 永久保留 |

## 資料結構

- `chat_rooms/{id}`:participantIds、lastMessage、unreadCount
- 訊息在**子集合** `chat_rooms/{id}/messages`(commit a721954 從頂層集合遷移,為了修 rules 權限問題)
- 訊息 update 一律禁止(rules 層 `allow update: if false`)
- 一對一聊天室由 App 端在 Match 結算時建立(rules:登入者可 create);群組聊天由 Cloud Function 建立

## 解鎖規則

- 見面前**沒有任何聊天入口**——聊天 Tab 空狀態顯示「互評解鎖」提示
- 群組聊天室內照片於週四 19:00 解鎖(見 [時間軸狀態機](時間軸狀態機.md) 照片規則);一對一聊天室直接顯示真實照片
- 禁止付費解鎖聊天(產品鐵律 2)

## 程式錨點

- `lib/providers/chat_provider.dart` — 訊息發送、聊天室列表
- `lib/models/chat_message_model.dart`
- `lib/screens/chat/`(chat_list_screen、chat_detail_screen)
- `functions/src/pushNotifications.ts` — 新訊息 trigger 推播
- 權限:`firestore.rules` 的 `isChatParticipant()` 與 messages 子集合的 `isChatMember()`(用 get() 查父文件)

## 踩過的雷

- 訊息放頂層集合時 rules 無法驗證參與者 → 遷移到子集合(a721954)
- 聊天輸入框排版/留白問題修過兩輪(41e90d4、49f409a)
