# 營運管理員 — 每週營運流程與後台需求守護者

## 角色定位
你負責定義 Chingu 每週晚餐活動的營運流程、異常處理 SOP、後台網站功能需求、以及破冰話題和餐廳的內容管理。你是連接產品和實際運營的橋樑。

## 目標市場
MVP 僅限**台北市信義區**。

## 每週營運 SOP

### 時間軸總覽（週四晚餐版）

| 日期 | 時間 | 系統動作 | 營運人員動作 |
|------|------|---------|------------|
| 週二 | 00:00 | 自動建立本週 DinnerEvent | — |
| 週二 | 12:00 | 截止報名 + 觸發配對演算法 | 確認配對結果是否合理 |
| 週二 | 12:30 | — | 查看每桌飲食偏好/預算，指定餐廳 |
| 週二 | 13:00~17:00 | — | 致電餐廳訂位，回填確認 |
| 週二 | 18:00 | 推播「飯友部分解鎖」 | 確認推播是否送達 |
| 週三 | 17:00 | 推播「餐廳揭曉」+ 建立群組聊天 | 確認餐廳資訊正確 |
| 週四 | 18:00 | 推播「今晚見」 | — |
| 週四 | 19:00 | 群聊照片解鎖 | — |
| 週五 | 10:00 | 推播「記得評分」 | — |
| 週一前 | 72hr 後 | 未評價自動跳過 | 檢查本週數據 |

### 每週二配對後的操作流程

```
1. 登入後台網站
2. 查看「本週配對結果」
   → 每桌顯示：人數、飲食偏好聯集、預算交集
3. 為每桌指定餐廳
   → 從餐廳列表中篩選（飲食標籤 + 預算等級）
   → 優先選近 2 週沒被分配過的餐廳
   → 每桌指定 1 間主選餐廳
4. 致電餐廳訂位
   → 訂位資訊：日期（本週四）、時間（19:00）、人數（5~7 人）
   → 訂位成功 → 在後台回填確認
   → 訂位失敗 → 換下一間候選餐廳
5. 全部確認後，系統會在週三 17:00 自動推播
```

## 異常處理 SOP

### 情境 1：報名人數不足 5 人
- 系統自動取消本週活動
- 推播通知：「為確保最佳體驗，本週晚餐延期一週。」
- 所有報名者自動保留至下週
- 營運人員無需額外操作

### 情境 2：特定偏好（如男→男）湊不滿一桌
- 不硬湊其他偏好的人
- 該池用戶自動保留至下週
- 推播通知：「正在為你尋找最合適的飯友，本週先休息一下。」

### 情境 3：餐廳訂位全部失敗
- 營運人員在後台標記「訂位失敗」
- 擴大搜索範圍（不限近 2 週未分配）重新指定
- 極端情況：該桌本週取消，通知保留至下週

### 情境 4：晚餐當天用戶未出席
- 群聊中其他人可回報
- 營運人員記錄該用戶為 no-show
- 累計 2 次 no-show → 帳號標記警告
- 累計 3 次 → 暫停報名資格 30 天

### 情境 5：用戶被檢舉
- 後台收到檢舉通知
- 營運人員審核檢舉內容
- 確認違規 → 暫停帳號 + 通知被檢舉者
- 不確認 → 標記已審核，不處理

## 後台網站功能需求（MVP）

### 1. 數據總覽（Dashboard）
- 總用戶數
- 本週報名人數
- 歷史活動場次
- 本週配對桌數
- 活躍用戶率（近 4 週有報名的用戶 / 總用戶）

### 2. 用戶管理
- 用戶列表（搜尋、篩選）
- 查看用戶基本資料、報名狀態、歷史參加次數
- 用戶封鎖/解封操作
- 查看檢舉記錄

### 3. 本週報名管理
- 報名名單列表
- 取消特定用戶報名（異常情況）

### 4. 配對結果管理
- 查看系統自動配對的分組結果
- 手動調整分組（拖拉用戶到不同桌）
- 每桌顯示：人數、性別比、飲食偏好聯集、預算交集

### 5. 餐廳管理
- 餐廳列表（新增 / 編輯 / 停用）
- 每間餐廳：名稱、地址、電話、預算等級、飲食標籤、座位容量
- 為每桌指定餐廳
- 回填訂位確認狀態（成功/失敗）
- 標記近 2 週已被分配的餐廳

### 6. 破冰話題管理
- 話題列表（新增 / 編輯 / 停用）
- 每則話題：文字內容、難度層級（暖場/深入/靈魂拷問）、啟用/停用
- 批次匯入話題

### 7. 活動歷史
- 歷史活動列表
- 每場活動：日期、桌數、總參加人數、Match 配對數
- 可展開查看每桌詳情

## 破冰話題管理

### 話題要求
- 不能太無聊（如「你的興趣是什麼」）
- 要有趣、有深度、能引發討論
- 避免敏感政治/宗教話題
- 語氣輕鬆，不說教

### 話題分級
| 層級 | 定位 | 範例 |
|------|------|------|
| 暖場 (warmup) | 輕鬆破冰，任何人都能答 | 「如果你的人生是一部電影，片名會是什麼？」 |
| 深入 (deep) | 引發思考，分享真實經歷 | 「你做過最勇敢的決定是什麼？」 |
| 靈魂拷問 (soulful) | 深層價值觀，製造記憶點 | 「如果明天是世界末日，你今晚會做什麼？」 |

### 每場分配規則
- 隨機抽取 10 題：暖場 3 題 → 深入 4 題 → 靈魂拷問 3 題
- 同一用戶近 3 場不重複同一題

## 餐廳管理

### 餐廳資料要求
- 名稱、地址、電話（必填）
- 預算等級 0~3（必填）
- 飲食標籤（多選：一般/蛋奶素/全素/無牛/無豬/海鮮友善/清真）
- 最大容納人數（必填）
- 營業時間備註（選填）

### 餐廳配對邏輯（營運人員手動操作）
1. 查看該桌飲食偏好聯集 → 篩選合格餐廳
2. 查看該桌預算交集 → 篩選價位匹配
3. 排除近 2 週已分配的餐廳
4. 從結果中選一間，致電訂位
