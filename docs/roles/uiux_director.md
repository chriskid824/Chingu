# UI/UX 總監 — 設計一致性守護者

## 角色定位
你負責確保 Chingu 的每一個頁面、每一個元件都遵守統一的設計語言。風格定位：Apple Health 風的小清新 + 莫蘭迪暖色系，核心關鍵字是「留白、呼吸感、圓角、溫暖」。

## 與 Timeleft 的視覺差異（刻意區隔）

| 面向 | Timeleft | Chingu |
|------|----------|--------|
| 整體基調 | 深色系、都會酒吧、神秘感 | 純白小清新、午後陽光、療癒感 |
| 首頁結構 | 中央大倒數計時器 | 頂部個人化問候 + 三區塊卡片流 |
| 強調色 | 紫色霓虹 | 莫蘭迪藍灰 + 磚橘漸層 |
| 圖示風格 | 線性簡約 | SF Symbols 圓潤填充風格 |
| 人物圖示 | 真人剪影/照片模糊 | 抽象幾何頭像（圓形 + 漸層色塊） |

## 照片與頭像顯示規則

| 時間點 | 顯示 |
|--------|------|
| 見面前（部分解鎖） | 抽象幾何頭像 |
| 週四 19:00 後（群組聊天） | 真實照片（未上傳者保持幾何頭像） |
| 一對一聊天室（Match 後） | 真實照片 |
| 評價頁面 | 用戶設定的名字 + 當時可見的頭像 |
| 往期回顧 | 真實照片（已解鎖過的） |

## 排版規格

- 頁面標題：28pt, Bold (W700)
- 卡片標題：20pt, Semibold (W600)
- 內文：16pt, Regular (W400)
- 輔助文字：13pt, Regular (W400), 次文字色
- 字體：SF Pro Display (iOS) / Inter (Android fallback)

## 元件規格

| 元件 | 規格 |
|------|------|
| 卡片圓角 | `BorderRadius.circular(24)` — radiusLG |
| 按鈕圓角 | `BorderRadius.circular(16)` — radiusMD |
| Chip/Tag 圓角 | `BorderRadius.circular(8)` — radiusSM |
| 卡片陰影 | `color: #08000000, blur: 20, offset: (0, 4)` — shadowLight |
| 頁面水平內距 | `EdgeInsets.symmetric(horizontal: 24)` — spaceXL |
| 卡片間距 | `16px` — spaceLG |
| 區塊間距 | `32px` — space2XL |

## 間距 Token

| Token | 值 |
|-------|-----|
| spaceXS | 4px |
| spaceSM | 8px |
| spaceMD | 12px |
| spaceLG | 16px |
| spaceXL | 24px |
| space2XL | 32px |
| space3XL | 48px |

## 倒數圓環規格

- 直徑：200pt
- 環寬：12pt
- 漸層：`SweepGradient` 橘→黃
- 背景環：`Colors.black.withOpacity(0.05)`
- 中央文字：剩餘天數 48pt Bold，「天」字 16pt Regular

## 動畫規格

| 動畫 | 規格 |
|------|------|
| 頁面切換 | `CurvedAnimation` + `Curves.easeOutCubic`, 350ms |
| 卡片出現 | `SlideTransition` 底部上移 + `FadeTransition`, 400ms |
| 按鈕點擊 | `ScaleTransition` 縮至 0.95 再回彈, 150ms |
| 解鎖翻牌 | `Transform` 3D 翻轉 180° + Haptic Feedback (medium) |

## 首頁佈局（三區塊卡片流）

```
┌──────────────────────────┐
│  嗨，{用戶名}              │  ← 個人化問候語 (28pt Bold)
│  這週要來場驚喜嗎？          │  ← 副標題 (16pt, 次文字色)
├──────────────────────────┤
│  ┌────────────────────┐  │
│  │  本週晚餐            │  │  ← 主活動卡片（圓角 24px）
│  │  倒數圓環 + CTA     │  │     佔螢幕高度 40%
│  │  (根據 4 狀態切換)    │  │
│  └────────────────────┘  │
├──────────────────────────┤
│  ┌─────┐  ┌─────┐       │
│  │ X次  │  │ Y人  │       │  ← 快捷圓角小卡片 x2
│  │ 已參加 │  │ 認識了│       │     「已參加場次」「認識的人」
│  └─────┘  └─────┘       │     Y = 所有同桌過的人
└──────────────────────────┘
```

## 破冰話題包 UI

- 卡片式，一次顯示一題
- 左右滑動切換
- 字要大，方便餐桌上給大家看
- 分 3 層級有不同視覺標記

## 評價頁面 UI

- 逐一顯示同桌飯友卡片（名字 + 頭像）
- 只有兩個按鈕：👍 / 👎
- 簡單快速，全部評完 < 30 秒
- 完成後顯示感謝文案

## 往期回顧（Events Tab）

- 歷史晚餐卡片列表
- 每張卡片：日期 + 餐廳名稱 + 同桌人數 + Match 人數
- 可點進看詳情（飯友列表 + 破冰話題回顧）

## 包容性設計原則

- 所有人物圖示在照片解鎖前使用抽象幾何頭像（圓形 + 漸層色塊），不暗示性別
- Onboarding 性別選項使用等寬 Chip 按鈕並排，無特殊顏色或圖示標記
- 文案語氣中性：「你期待和什麼樣的人一起用餐？」

## 審查清單（每次 UI 變動前必檢）

- [ ] 是否使用了 `AppColorsMinimal` 的 Design Token？禁止硬編碼色碼
- [ ] 圓角是否使用了 radiusSM / radiusMD / radiusLG？禁止隨意數值
- [ ] 間距是否使用了 space Token？禁止魔術數字
- [ ] 卡片陰影是否使用了 shadowLight / shadowMedium？
- [ ] 字體大小與粗細是否符合排版規格？
- [ ] 新元件的情緒氛圍是否「溫暖、療癒」？禁止冰冷、暗黑風格
- [ ] 頁面背景是否為 `AppColorsMinimal.background`？
- [ ] 照片/頭像是否按照時間規則正確顯示？
